Guía de Implementación Técnica: Marketplace de Turismo (Nuxt.js)

Para asegurar que los diseños se traduzcan fielmente a tu aplicación Nuxt, he preparado estas directrices organizadas por capas.

1. Configuración de Estilos y Tematización

Utiliza Tailwind CSS para una implementación rápida y coherente con los diseños:

  • Colores: Define el azul turístico como primary: #0077cc en tu tailwind.config.js.
  • Tipografía: Configura Inter como la fuente principal para mantener la legibilidad moderna.
  • Bordes: Aplica un radio de curvatura estándar de 12px (rounded-xl en Tailwind) para botones y tarjetas.

2. Arquitectura de Componentes (Atomic Design)

Divide la interfaz en componentes reutilizables:

  • UI Base: VButton, VCard, VBadge (para "Best Seller"), VInput, VRating.
  • Bloques: TourCard, SearchNavbar, FilterSidebar, BookingWidget.
  • Secciones: HeroSection, ReviewList, ImageGallery.

3. Estrategia de Fetching de Datos (Nuxt 3)

Para maximizar el SEO en las páginas de destino y detalles:

  • useAsyncData o useFetch: Implementa estas funciones en la página de resultados y detalle del tour para renderizado del lado del servidor (SSR).
  • Estado Global: Utiliza Pinia para gestionar el carrito de compra y la información del usuario durante el flujo de checkout.

4. Detalles de Implementación por Página

Home & Resultados

  • Imágenes: Usa el componente <NuxtImg> para optimización automática, lazy loading y formatos modernos (WebP).
  • Filtros: Implementa los filtros mediante parámetros en la URL (Query Params).

Detalle del Tour

  • Sticky Sidebar: Asegura que el widget de reserva use position: sticky con un margen superior adecuado.
  • Galería: Considera usar una librería liviana para el modal de fotos o impleméntalo con transiciones nativas de Vue.

5. UX & Micro-interacciones

  • Hover States: Añade transiciones suaves (transition-all duration-300) en las TourCard.
  • Loading States: Implementa esqueletos de carga (Skeletons) mientras se obtienen los resultados.
